Git & GitHub for Beginners

Your Safety Net and Collaboration Hub

Git vs. GitHub

  • Git: A version control system on your local machine. It tracks changes and creates snapshots of your code.
  • GitHub: A web-based platform that hosts your Git repositories. It's for remote storage, collaboration, and project management.

The 3-Stage Local Workflow

How changes are saved on your machine.

  1. Working Directory: The files you are currently editing.
  2. Staging Area: Where you place the changes you want to include in your next snapshot.
  3. Local Repository: Where Git permanently stores your snapshots (commits).

Command 1: `git add`

Moves changes from the Working Directory to the Staging Area.


# Stage a single file
$ git add index.html

# Stage all changes in the current directory
$ git add .

Command 2: `git commit`

Takes everything from the Staging Area and creates a permanent snapshot in your Local Repository.


$ git commit -m "feat: Add dark mode toggle button"

Always write a clear, descriptive commit message!

The Remote Workflow

Syncing your local repository with a remote one (GitHub).

  • `git push`: Sends your local commits to the remote repository.
  • `git pull`: Fetches changes from the remote repository and merges them into your local repository.

The Essential Daily Workflow


# 1. Get the latest changes from the remote
$ git pull

# 2. Make your changes to the code...

# 3. Stage your changes
$ git add .

# 4. Commit your changes
$ git commit -m "Your descriptive message"

# 5. Push your changes to the remote
$ git push

Key Takeaway

`add` -> `commit` -> `push`. This is the core loop for saving and sharing your work.
