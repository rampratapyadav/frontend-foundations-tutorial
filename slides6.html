Introduction to TypeScript

Supercharging Your JavaScript

What is TypeScript?

TypeScript is a "superset" of JavaScript. It's just JavaScript + a static type system.

Its goal is to catch errors before you run your code.

The Problem with JavaScript

JavaScript is dynamically typed. You don't know about type errors until you run the code.


function calculateTotal(item) {
  return item.price * item.quantity;
}

// This will result in NaN (Not a Number)
// You won't know until this code runs in the browser!
calculateTotal({ cost: 10, amount: 2 });

The TypeScript Solution: Interfaces

We can define the "shape" of our data with an `interface`.


interface Item {
  price: number;
  quantity: number;
}

function calculateTotal(item: Item): number {
  return item.price * item.quantity;
}

// Now, TypeScript gives an error right in your editor!
// ‚ùå Error: Property 'price' is missing.
calculateTotal({ cost: 10, amount: 2 });

The Compilation Step

Browsers don't run TypeScript. You must compile it to JavaScript first.

1. You write code in app.ts

2. You run the compiler: `npm run build` (which runs `tsc`)

3. The compiler checks for errors.

4. It creates a new file: dist/app.js

5. Your HTML loads the compiled JavaScript: `<script src="dist/app.js"></script>`

Key Takeaway

TypeScript helps you catch bugs before they happen by adding static types to JavaScript.
