Accessibility (A11y) Fundamentals

Building for Everyone

The POUR Principles

The foundation of web accessibility (WCAG).

  • Perceivable: Information must be presentable to users in ways they can perceive.
  • Operable: User interface components and navigation must be operable.
  • Understandable: Information and the operation of user interface must be understandable.
  • Robust: Content must be robust enough to be interpreted by a wide variety of user agents, including assistive technologies.

1. Semantic HTML

The foundation of accessibility. Use the right tag for the right job.



<nav>
  <a href="/">Home</a>
</nav>


<div>
  <a href="/">Home</a>
</div>

2. Keyboard Navigation

All interactive elements must be reachable and usable with only a keyboard.

  • Use `<button>` for actions, `<a>` for links.
  • Never remove the focus outline without a clear alternative.

/* Don't do this! */
:focus {
  outline: none;
}

/* Do this instead */
:focus-visible {
  outline: 3px solid blue;
}

3. ARIA Attributes

Accessible Rich Internet Applications. Use when HTML isn't enough.


<!-- Gives a button a descriptive name for screen readers -->
<button aria-label="Close dialog">X</button>

<!-- Tells screen readers to announce changes to this area -->
<div id="status" aria-live="polite"></div>

4. Color Contrast

Text must have sufficient contrast with its background. (Ratio of at least 4.5:1)

Good Contrast (4.53:1)

Bad Contrast (2.23:1)

Key Takeaway

Accessibility is not a feature, it's a fundamental requirement. Build with empathy.
