Frontend Architecture & System Design

Building to Scale

The Challenge: State Management

As apps grow, sharing data between many components becomes complex.

The naive approach, passing props down through many layers, is called "Prop Drilling" and is hard to maintain.

Solution: Centralized State Management

Libraries like Redux, Zustand (for React), or Pinia (for Vue) provide a central "store".

  • All shared application state lives in one place.
  • Components dispatch "actions" to update the state.
  • Components subscribe to changes and automatically re-render.

API Layer Design: REST vs. GraphQL

How your frontend talks to your backend.

REST

Multiple requests to different endpoints to gather all needed data. Can lead to over- or under-fetching.


GET /users/1
GET /users/1/posts
GET /users/1/comments

GraphQL

A single, flexible query to get exactly the data you need in one request.


query {
  user(id: 1) {
    name
    posts {
      title
      comments { body }
    }
  }
}

Key Takeaway

For large apps, use architectural patterns like centralized state management and choose an API strategy (like GraphQL) that fits your needs.
