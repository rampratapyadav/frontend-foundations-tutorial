Prompt Engineering for Code

Getting What You Want from AI

The Anatomy of a Perfect Prompt

A great prompt is a detailed spec sheet for your AI assistant.

  1. Role & Goal: Tell the AI who it is. ("You are an expert TypeScript developer...")
  2. Task & Action: State the specific task. ("Write a function that...")
  3. Context & Constraints: Provide tech stack, existing code, and limitations.
  4. Format & Structure: Specify the desired output format. ("Return only the code...")
  5. Examples: Provide clear input/output examples.

Example: From Bad to Great

Task: Convert an array of objects to an object keyed by ID.

Bad Prompt: "write a function that converts an array to an object"

(Too vague, AI has to guess)

Great Prompt: "You are an expert TypeScript developer. Write a generic function `arrayToObject` that takes an array of objects and a key, and returns an object keyed by that property. Handle empty arrays. Provide an input/output example. Return only the function with JSDoc comments."

(Specific, contextual, and provides examples)

A Great Prompt Example


You are an expert TypeScript developer.

Task: Write a function named `arrayToObject`...

Constraints: Must be generic, handle empty arrays, use no external libraries.

Example:
Input: `const users = [{ id: 'a', name: 'Alice' }];`
Output: `{ 'a': { id: 'a', name: 'Alice' } }`

Format: Provide only the TypeScript function, with JSDoc comments.

Key Takeaway

The quality of your AI's output is a direct reflection of the quality of your prompt.
